# Live Coding Interview 练习仓库

用于准备和练习技术面试中的 live coding 环节。

## 仓库结构

```
company/
└── {公司名}/
    └── {面试轮次}/
        ├── instructions.md    # 面试说明
        ├── *-prep.md         # 准备笔记
        └── *.py / *.js / ... # 实际练习代码
```

## 使用方式

### 1. 按公司组织面试准备

每次收到新的面试邀请时：

```bash
mkdir -p "company/{公司名}/{轮次名称}"
```

将面试说明、准备笔记和练习代码都放在对应目录下。

### 2. 练习常见题型

常见的 live coding 题型：

- **Toy Robot** - 经典 OOP 练习，涉及状态管理、命令解析、边界检查
- **API 设计** - RESTful 端点设计、数据建模、错误处理
- **数据结构实现** - 栈、队列、树、图的基本操作
- **算法优化** - 时间/空间复杂度优化、缓存策略

### 3. 本地环境准备

确保以下工具可用：

- Python 3 / Node.js / Java / 你选择的语言环境
- 测试框架（pytest / jest / junit）
- Git 和 GitHub CLI（用于 PR 流程面试）
- 编辑器和终端熟练度

## 面试最佳实践

### 沟通节奏

- **先理解再动手** - 确认需求边界，口头总结理解
- **边写边说** - 说出思路和权衡，不要长时间沉默
- **增量开发** - 先跑通最小版本，再迭代优化

### 代码质量

- 变量命名清晰（`new_position` 优于 `np`）
- 函数职责单一
- 写完立即验证，不要累积错误
- 简单优于复杂，避免过度设计

### 常见陷阱

- ❌ 过度设计 - 不需要一开始就上设计模式
- ❌ 沉默写代码 - 面试官看不到你的思考过程
- ❌ 忽略现有代码 - 要先理解已有结构
- ❌ 不跑测试 - 边写边验证很重要

## Git 分支策略

建议为每个公司的面试流程创建独立分支：

```bash
git checkout -b interview/{公司名}-{日期}
```

面试结束后可以将准备过程和练习代码合并回 main，作为日后参考。

---

**提示**：面试不是考试，是和未来同事一起解决问题。保持放松，展现真实的工作方式。
