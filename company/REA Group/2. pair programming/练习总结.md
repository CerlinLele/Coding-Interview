## 完成的功能

### 1. BACKWARD 命令

**需求**：
- 机器人向当前朝向的反方向移动一步
- 朝向保持不变
- 需要遵守边界检查（不能掉出网格）

**实现**：
```python
def backward(self):
    if not self.is_placed():
        return False
    
    dx, dy = self.DIRECTION_DELTAS[self.facing]
    new_x = self.x - dx  # 反向移动
    new_y = self.y - dy
    
    if self.table.is_valid_position(new_x, new_y):
        self.x = new_x
        self.y = new_y
        return True
    
    return False
```

**验证**：通过手动测试确认功能正常

---

### 2. UNDO 命令（进行中）

**需求**：
- 撤销上一步状态改变操作
- 支持 MOVE、BACKWARD、LEFT、RIGHT
- 不支持撤销 PLACE、REPORT
- 支持多次连续 UNDO

**设计决策**：
1. **状态存储**：保存完整的 `(x, y, facing)` 三元组
   - 原因：既有位置变化（MOVE/BACKWARD）也有朝向变化（LEFT/RIGHT）
2. **数据结构**：使用 list 作为 FILO 栈
3. **保存时机**：在状态改变操作执行前保存当前状态
4. **恢复机制**：pop 最后保存的状态并恢复

## 踩过的坑

### 坑 1：类变量 vs 实例变量

**问题**：
```python
class Robot:
    HISTORY = []  # ❌ 类变量，所有实例共享
    
    def __init__(self, table):
        self.table = table
        # 缺少 self.history = []
```

**现象**：
- 所有 Robot 实例共享同一个 `HISTORY` 列表
- 测试之间互相污染，`test_undo_without_placement` 检查空列表时发现有数据

**原因**：
- 类变量定义在 `class` 块内，`__init__` 外
- 所有实例通过 `self.HISTORY` 访问的是同一个对象

**修复**：
```python
def __init__(self, table):
    self.table = table
    self.x = None
    self.y = None
    self.facing = None
    self.history = []  # ✅ 实例变量
```

**教训**：
- Python 类变量和实例变量的区别
- 可变对象（list/dict）作为类变量特别危险
- 每个实例的独立状态必须在 `__init__` 中初始化

---

### 坑 2：变量名大小写不一致

**问题**：
- `__init__` 中定义：`self.history = []`（小写）
- `undo()` 中使用：`self.HISTORY`（大写）
- 结果：`AttributeError: 'Robot' object has no attribute 'HISTORY'`

**原因**：
- Python 区分大小写
- 混用导致引用不到实际变量

**修复**：
- 统一使用 `self.history`（小写，实例变量）
- 类常量使用大写（如 `DIRECTIONS`、`DIRECTION_DELTAS`）

**教训**：
- 保持命名约定一致性
- 类常量用大写，实例变量用小写

### 坑 3：保存历史的时机错误（当前问题）

**问题**：
```python
def execute(self, command):
    # ❌ 无条件保存所有命令的状态
    self.history.append((self.x, self.y, self.facing))
    
    command = command.strip().upper()
    
    if command == 'MOVE':
        self.move()
    elif command == 'UNDO':
        return self.undo()
```

**执行流程**：
1. `PLACE 0,0,NORTH` → 保存 `(None, None, None)` → 放置到 (0,0,NORTH)
2. `MOVE` → 保存 `(0,0,NORTH)` → 移动到 (0,1,NORTH)
3. `UNDO` → **先保存 `(0,1,NORTH)`** → pop 出 `(0,0,NORTH)` → 恢复到 (0,0,NORTH)

**结果**：
- UNDO 把自己刚保存的状态 pop 出来了
- 测试期望 `0,0,NORTH`，实际还是 `0,1,NORTH`

**根本原因**：
- UNDO 命令本身不应该保存历史
- PLACE 和 REPORT 也不应该保存历史
- 只有状态改变操作（MOVE/BACKWARD/LEFT/RIGHT）才需要保存

**正确方案**：
- 删除 `execute()` 中的无条件保存
- 在 `move()`、`backward()`、`left()`、`right()` 各方法内部，执行前保存状态
- UNDO 直接 pop 并恢复，不保存

**教训**：
- 需要精确控制"在哪个命令执行前"保存状态
- 不是所有命令都需要历史记录
- UNDO 这种元操作本身不应被记录

## 第二轮练习（2026-06-26）

### 完成的功能

#### 3. UNDO 重构 — 历史保存移入各方法

**问题背景**：原来在 `execute()` 中统一保存历史，现重构为各方法内部自己负责。

**设计决策**：
- 历史保存在 `is_placed()` 检查之后
- `move()` 和 `backward()` 仅在新位置合法时保存（失败的移动不记录）
- `left()` 和 `right()` 在操作执行前保存（旋转总是成功）

**最终实现模式**：
```python
def move(self):
    if not self.is_placed():
        return False
    dx, dy = self.DIRECTION_DELTAS[self.facing]
    new_x, new_y = self.x + dx, self.y + dy
    if self.table.is_valid_position(new_x, new_y):
        self.history.append((self.x, self.y, self.facing))  # 成功才保存
        self.x, self.y = new_x, new_y
        return True
    return False
```

#### 4. MOVE_COUNT 功能

**需求澄清（面试沟通过程）**：
| 问题 | 决策 | 理由 |
|------|------|------|
| BACKWARD 算 move 吗？ | 算 | 位置发生了变化 |
| count 存在哪里？ | `self.move_count` 独立变量 | 与 history 职责分离 |
| UNDO 要减 count 吗？ | 要 | 保持与 history 一致性 |
| PLACE 重置 count 吗？ | 不重置 | PLACE 是初始化，不是状态变化 |

**UNDO 判断是否减 count 的方法**：
- pop 出旧状态后，比较 `current_state[2]`（旧 facing）与 `self.facing`（当前 facing）
- 相等 → 撤销的是移动操作 → 减 count
- 不等 → 撤销的是旋转操作 → 不减 count

```python
def undo(self):
    if not self.history:
        return False
    current_state = self.history.pop()
    if current_state[2] == self.facing:
        self.move_count -= 1
    self.x, self.y, self.facing = current_state
    return True
```

**REPORT 新格式**：`x,y,FACING,count`
