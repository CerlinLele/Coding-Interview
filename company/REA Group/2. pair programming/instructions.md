Our engineering recruitment philosophy

We use the following backend technologies at REA: 

Java (Version 11)  
Ruby 
Scala 
Typescript (Node.js version 16) 
Node.js 
Python (Version 3) 
.Net (Version 6)  

Our engineers work across these daily, some more specialised than others, but all pragmatic and language agnostic. We believe software is a tool to create great products and experiences.  
This stage should last approximately 90 Minutes. We will be providing you a completed code asset in a programming language of your choice, and you'll be asked to build an extension. The language choices are Python, Scala, Java, Typescript, .Net or Ruby. We will be sharing the code base with you at the beginning of your interview.  

If you haven't already provided this to us, please email a shared GitHub account user name and programming language preference to talentacquisition@rea-group.com. Ensure this is a Github account you are comfortable having added to our REA repository for the purposes of this assessment.  

Please ensure that you have the following ready to go before the interview commences:  

A Github account and ability to clone a repository in your machine 
A code editor that you prefer 
The compiler of the language that you chose and ability to run tests and code  
We’re assessing you on three dimensions:  

Current Technical Knowledge – What do you know and what are your engineering skills?  
Pragmatism – Simple is often better; we don’t like to over complicate solutions
Learning Mindset – We are always learning at REA. We’re open to ideas and we love to share!  
While the code base we are assessing is in one of the above languages, we are not assessing your knowledge or capability of that language. 

Good luck, and please get back to us if you have any questions!
